## 3. 핵심 기능 분석

### 3.1 빌드 시스템 (`build_bali.py`)
- **기능**: BALI_DATA 딕셔너리 기반 924개 HTML 페이지 자동 생성
- **SEO**: og:title, og:description, og:image, canonical URL, schema.org Article 마크업
- **이미지 배분**: Fisher-Yates 시드 셔플 방식으로 페이지별 고유 이미지 10장 배치
- **콘텐츠**: Q&A 서론, 카테고리별 맛집/명소, 가격비교표, 교통정보, 숙소추천, 실전팁
- **제휴**: 마이리얼트립 할인쿠폰 하이퍼링크 (상단+하단)
- **생성일**: 페이지별 고유 날짜 (2026-04-01 기준)

### 3.2 이미지 시스템
- **매핑**: `image_mapping_v3.json` — 지역×카테고리별 WEBP 파일명 리스트
- **중복제거**: perceptual hash (8×8 그레이스케일 → 64비트 해시, 해밍거리 ≤5 중복 판정)
- **이미지 소스**: Picsum Photos (ID 기반, 1084개 고유 사진)
- **형식**: WebP, 품질 80, 최소 1200×800

### 3.3 스크래퍼 시스템
- **소스**: Openverse API, Flickr 공개 피드, Wikimedia Commons, Picsum (fallback)
- **방식**: 키워드 기반 검색 → 다운로드 → perceptual hash 중복검사 → WebP 변환 → 매핑 업데이트
- **변형**: v5(기본), expanded(확장), mega(대용량), multi(멀티소스), unique(고유전용)

### 3.4 SEO 시스템 (`seo_titles.py`)
- **제목**: 지역+카테고리+페이지번호+연도 조합 (중복 방지)
- **메타 설명**: 150자 이내, 핵심 키워드 포함
- **키워드**: 지역명, 카테고리, 발리, 인도네시아 등

## 4. 발견된 이슈 및 해결

### 4.1 이미지 부족 (해결 ✅)
- **문제**: 20개 지역/카테고리 조합이 14페이지에 필요한 이미지 14개 미만 (최소 9개)
- **영향**: 로비나(5개 카테고리), 베두굴(3개), 짠디다사(2개), 킨타마니(4개), 타나롯(5개)
- **해결**: fill_shortages.py + fast_expand 시리즈로 4,245개 신규 다운로드
- **결과**: 모든 조합 119개 이상 확보

### 4.2 페이지 간 이미지 중복 (해결 ✅)
- **문제**: 같은 지역/카테고리 14페이지가 동일한 이미지 10장을 공유
- **원인**: `get_images()` 함수가 단순 `[:count]` 슬라이싱
- **해결**: Fisher-Yates 시드 셔플 방식 도입 (페이지 인덱스 기반 고유 시드)
- **결과**: 924페이지 모두 고유 이미지 세트 확인

### 4.3 카테고리 간 이미지 중복 (해결 ✅)
- **문제**: 같은 지역 내 다른 카테고리에 동일 이미지가 매핑될 수 있음
- **해결**: perceptual hash 기반 전역 중복 검사
- **결과**: 매핑 내 크로스카테고리 중복 0건

### 4.4 SEO 메타 중복 (기존 해결)
- **문제**: og:image가 모든 페이지에서 동일
- **해결**: 지역별 대표 이미지 매핑
