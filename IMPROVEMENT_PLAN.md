[Phase 1] 긴급 수정 (즉시 적용)
============================================================

1-1. 중복 이미지 교체 (37개)
  대상: Perceptual hash 해밍 거리 ≤5인 이미지 쌍
  방법:
    - 중복 이미지 삭제 후 해당 카테고리에서 재다운로드
    - Openverse에서 다른 키워드 조합으로 검색
    - Picsum 시드 변경 (현재 시드에서 +100 오프셋)
  예상 소요: 30분
  명령어:
    python3 scraper_multi.py --fix-duplicates

1-2. 저해상도 이미지 교체 (3개)
  대상:
    - 타나롯/transport/타나롯_transport_0001.webp (1022×179)
    - 타나롯/transport/타나롯_transport_0008.webp (1022×179)
    - 타나롯/transport/타나롯_transport_0010.webp (1022×179)
  방법:
    - 최소 800×600 이상 이미지로 교체
    - "bali road" 또는 "bali transport" 키워드로 재검색
  예상 소요: 10분

1-3. 중국어 문장 제거
  대상: 스미냑 Q&A ("哪家最好?")
  방법:
    - build_bali.py의 BALI_DATA에서 중국어 문장 한국어로 대체
    - 예: "哪家最好?" → "어떤 곳이 가장 좋을까요?"
  예상 소요: 5분

1-4. 이미지 경로 최종 검증
  방법:
    - 빌드 후 HTML에서 이미지 경로가 실제 파일과 매칭되는지 검증
    - 자동 검증 스크립트 추가
  명령어:
    python3 -c "from build_bali import *; verify_image_paths()"

[Phase 2] 콘텐츠 품질 향상 (1~2주)
============================================================

2-1. 페이지별 콘텐츠 다양화
  현재 문제:
    - 같은 카테고리의 14페이지가 거의 동일한 구조
    - food의 음식 목록이 6개 고정, 페이지별 2개씩 순환
  개선 방안:
    - 각 페이지마다 고유한 도입부 문단 작성
    - 음식 목록을 15~20개로 확장, 페이지별 4개씩 랜덤 선택
    - "숨은 맛집", "현지인 추천", "가성비 BEST" 등 테마 부여
    - 계절별 추천 콘텐츠 (우기/건기 다른 추천)
  예상 효과: 페이지 고유성 ↑, SEO 개선

2-2. 빈 카테고리 보강
  현재 문제:
    - 스미냑/nature: 데이터 1개
    - 누사두아/nature: 데이터 0개
    - transport 카테고리 전반적으로 데이터 적음
  개선 방안:
    - 스미냑: 페티탕겟 해변 산책로, 스미냑 정원 등 추가
    - 누사두아: 워터블로우 자연 명소, 누사두아 정원 등 추가
    - transport: 지역별 상세 교통 가이드 확장
  예상 소요: 2~3시간

2-3. 이미지 소스 다양화
  현재: Openverse + Flickr + Wikimedia + Picsum
  추가 계획:
    - Unsplash API 키 발급 (무료 50 req/hr)
      → https://unsplash.com/developers
    - Pexels API (무료 200 req/hr)
      → https://www.pexels.com/api/
    - 직접 촬영 이미지 업로드
  예상 효과: 이미지 품질 ↑, 중복 ↓

2-4. SEO 제목 고유성 강화
  현재: 913/924 고유 (98.8%)
  목표: 100% 고유
  방법:
    - 템플릿 추가 (현재 8개 → 15개)
    - 키워드 조합 확장
    - 페이지 번호 접미사 적용 (수정 완료)

[Phase 3] 시스템 개선 (2~4주)
============================================================

3-1. 이미지 자동 분류 시스템
  도구: CLIP (OpenAI) 또는 BLIP
  방법:
    - 다운로드된 이미지에 CLIP 적용
    - 카테고리별 confidence score 계산
    - 0.7 미만 이미지는 수동 검토 큐에 추가
  예상 소요: 1일 (구현) + ongoing

3-2. Selenium 기반 스크래퍼
  대상: Ctrip, 마펑우(马蜂窝)
  도구: Selenium + Chrome Headless
  방법:
    - 모바일 User-Agent로 접근
    - 이미지 URL 추출 후 다운로드
    - 카테고리별 자동 분류
  제약: 서버에 Chrome 설치 필요
  예상 소요: 2~3일

3-3. 사이트맵/robots.txt 자동 생성
  방법:
    - build_bali.py에 sitemap.xml 생성 로직 추가
    - 924개 페이지 + 660개 이미지 URL 포함
    - robots.txt 작성
  예상 소요: 2시간

3-4. 내부 링크 강화
  현재: 관련 지역 추천 3개
  개선:
    - 같은 카테고리 다른 지역 링크
    - 같은 지역 다른 카테고리 링크
    - 인기 페이지 TOP10 링크
    - "이 글과 비슷한 글" 추천 알고리즘
  예상 소요: 1일

3-5. 에러 핸들링 강화
  현재 문제:
    - API 실패 시 복구 로직 부족
    - 이미지 다운로드 실패 시 재시도 없음
  개선:
    - exponential backoff 재시도
    - 실패 로그 파일 생성
    - 상태 파일 자동 복구
  예상 소요: 반일
